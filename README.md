# インターン選考管理 PWA（v10）

大学生が、インターンシップや選考中の企業情報をブラウザ上で整理するための、静的Webアプリです。

GitHub Pagesで公開して利用できます。サーバーやデータベースは使用せず、登録データは利用しているブラウザ内に保存されます。

---

## 1. このアプリでできること

- 社名だけで企業情報を登録する
- 選考状況を記録する
- 希望度を記録する
- ES締切、Webテスト締切、適性検査締切などを複数登録する
- インターン日時、面接日時を記録する
- 企業のマイページURLを保存して開く
- 未登録企業について「社名 公式サイト」でWeb検索する
- ログインID・パスワードをアプリ用パスワードで暗号化して保存する
- 保存済みのID・パスワードをコピーする
- GitHub PagesでPWAとして公開し、スマートフォンやPCで利用する

> 卒業年度は入力しません。社名だけが必須で、それ以外の項目は未定・未入力のまま登録できます。

---

## 2. 主な画面

| 画面・エリア | 内容 |
|---|---|
| 上部ヘッダー | ネットワーク状態の表示、認証情報のロック解除・ロック、更新反映ページへのリンク |
| 企業検索 | 登録済み企業なら保存済みマイページを開き、未登録企業なら公式サイト検索を開く |
| 企業登録フォーム | 社名、選考状況、希望度、締切、日時、URL、認証情報、メモを入力する |
| 登録済み企業一覧 | 保存した企業情報の確認、編集、削除、URL・認証情報のコピーを行う |
| 認証情報ダイアログ | アプリ用パスワードの初回設定、またはロック解除を行う |
| 更新反映ページ | 古いPWAキャッシュとService Workerを削除して最新版を読み込む |

---

## 3. 登録できる情報

| 項目 | 必須 | 内容 |
|---|---|---|
| 社名 | 必須 | 100文字まで入力できます |
| 選考状況 | 任意 | 応募予定、応募済み、書類選考中、面接予定、合格、不合格、辞退など |
| 希望度 | 任意 | 第一志望、高い、普通、低い |
| 締切 | 任意 | 種類、締切名、締切日時を複数登録できます |
| インターン日時 | 任意 | 日時が未定の場合は空欄で登録できます |
| 面接日時 | 任意 | 日時が未定の場合は空欄で登録できます |
| 企業マイページURL | 任意 | `http://` または `https://` で始まるURLを入力します |
| ID | 任意 | 認証情報をロック解除した状態で入力できます |
| パスワード | 任意 | 認証情報をロック解除した状態で入力できます |
| メモ | 任意 | 提出物、連絡事項、選考メモなどを入力できます |

## 4. 使い方

### 4-1. 企業を登録する

1. 「企業を登録」フォームの社名を入力します。
2. 必要に応じて選考状況、希望度、締切、日時、マイページURL、メモを入力します。
3. 「登録する」を押します。
4. 登録済み企業一覧に企業が表示されます。

社名だけを入力して登録することもできます。

### 4-2. 締切を追加する

1. 「締切」エリアに最初から表示されている入力行へ、種類・締切名・締切日時を入力します。
2. 締切を増やす場合は「締切を追加」を押します。
3. 追加された行に次の締切を入力します。
4. 不要な締切行は、その行の「削除」を押します。

締切日時が未定の場合は、種類または締切名だけ入力して登録できます。

### 4-3. 企業のサイトを開く

1. 上部の企業検索欄に社名を入力します。
2. 「サイトを開く」を押します。

動作は次のとおりです。

| 状態 | 動作 |
|---|---|
| 登録済みでマイページURLあり | 保存済みの企業マイページを新しいタブで開く |
| 登録済みだがマイページURLなし | 「社名 公式サイト」で検索する |
| 未登録の企業 | 「社名 公式サイト」で検索する |

### 4-4. 認証情報を使う

初めて認証情報を保存する場合は、右上の「認証情報をロック解除」を押します。

1. 8文字以上のアプリ用パスワードを入力します。
2. 確認用パスワードにも同じ文字列を入力します。
3. 「設定して続ける」を押します。
4. ID・パスワード欄が入力できる状態になります。
5. 企業情報を登録または更新すると、ID・パスワードは暗号化して保存されます。

次回以降は、右上の「認証情報をロック解除」からアプリ用パスワードを入力します。

> アプリ用パスワードを忘れると、保存済みのID・パスワードを復元できません。必ず自分で安全に管理してください。

### 4-5. ID・パスワードをコピーする

1. 右上の「認証情報をロック解除」を押してロックを解除します。
2. 一覧の対象企業で「IDコピー」または「パスコピー」を押します。
3. 必要な入力欄へ貼り付けます。

ロック中は認証情報のコピーはできません。

## 5. 保存先とセキュリティ

### データの保存先

企業情報は、利用しているブラウザの `localStorage` に保存されます。

そのため、次の点に注意してください。

- 別のPC・別のスマートフォン・別のブラウザには自動同期されません。
- ブラウザのデータ削除、サイトデータ削除、シークレットモード終了などでデータが失われる場合があります。
- 同じ端末・同じブラウザで利用してください。
- 重要な情報は別途バックアップを取ってください。

### 認証情報の扱い

ID・パスワードは、アプリ用パスワードから生成した鍵で暗号化してブラウザ内へ保存します。

ただし、次の制約があります。

- アプリ用パスワードの復元機能はありません。
- 公共PCや他人と共有する端末での保存は推奨しません。
- GitHubリポジトリ、HTML、JavaScript、READMEに実際のID・パスワードを書かないでください。
- このアプリは企業サイトへの自動ログイン、応募、情報取得は行いません。

## 6. GitHub Pagesで公開する方法

### 6-1. GitHubリポジトリを作成する

1. GitHubへログインします。
2. 右上の `+` から `New repository` を選びます。
3. リポジトリ名を入力します。例：`inta-n`
4. `Public` を選びます。
5. `Create repository` を押します。

### 6-2. ファイルをアップロードする

リポジトリのトップ画面で、`Add file` → `Upload files` を選びます。

次のファイルとフォルダを、リポジトリの一番上の階層へアップロードします。

```text
index.html
styles.css
app.js
sw.js
manifest.webmanifest
reset.html
README.md
icons/
  icon.svg
```

アップロード後、`Commit changes` を押します。

> `intern-selection-manager-v10` のようなフォルダごとアップロードして、ファイルが二重の階層にならないよう注意してください。

### 6-3. GitHub Pagesを有効にする

1. リポジトリの `Settings` を開きます。
2. 左メニューの `Pages` を開きます。
3. `Build and deployment` の `Source` で `Deploy from a branch` を選びます。
4. Branchは `main`、フォルダは `/(root)` を選びます。
5. `Save` を押します。

公開URLは、通常は次の形式です。

```text
https://GitHubユーザー名.github.io/リポジトリ名/
```

例：

```text
https://csya24102-ai.github.io/inta-n/
```

## 7. アプリを更新する方法

### 通常の更新

HTML、CSS、JavaScriptを更新した場合は、GitHub上で対象ファイルを開き、鉛筆アイコンから内容を置き換えて `Commit changes` を押します。

### 重要：PWA更新時にそろえるもの

このアプリはPWAのキャッシュを利用します。更新時は、次の番号を同じ値にそろえてください。

| ファイル | 更新する箇所 |
|---|---|
| `sw.js` | `const CACHE_NAME = "intern-selection-manager-v10";` |
| `index.html` | `styles.css?v=10`、`app.js?v=10`、`manifest.webmanifest?v=10` |
| `app.js` | `register("./sw.js?v=10", ...)` |
| `manifest.webmanifest` | `start_url` の `?v=10` |
| `reset.html` | 最新版へ戻すURLの `?v=10` |

次の更新でv11にする例です。

```text
v10 → v11
```

`sw.js` のキャッシュ名、`index.html` のCSS・JavaScript読み込み番号、`app.js` のService Worker読み込み番号などを、すべて `v11` にそろえます。

### 更新が反映されない場合

公開URLを通常のブラウザタブで開き、右上の「更新が反映されない場合」を押します。

または、次のURLを開きます。

```text
https://GitHubユーザー名.github.io/リポジトリ名/reset.html?v=10
```

表示されたページで「古いPWAキャッシュを削除して最新版を開く」を1回押します。

この処理は、古いService WorkerとCache Storageを削除して最新版を読み込むためのものです。

> 「ブラウザのサイトデータを削除」「Clear site data」は、登録した企業情報まで消える可能性があるため、通常は押さないでください。

## 9. 動作確認の目安

アプリを開いたら、まず右上の状態表示を確認します。

| 表示 | 意味 |
|---|---|
| オンライン | JavaScriptが起動し、ネットワーク接続中です |
| オフライン | JavaScriptは起動していますが、ネットワーク未接続です |
| 確認中 | ページ読み込み直後の初期表示です。長時間変わらない場合はJavaScriptまたはキャッシュに問題がある可能性があります |
| 起動エラー | JavaScriptの起動中にエラーが発生しています。最新版の `index.html`、`app.js`、`sw.js` が揃っているか確認してください |

次の3点が動けば、基本機能は正しく起動しています。

1. 「認証情報をロック解除」を押すと、アプリ用パスワードの画面が開く
2. 「締切を追加」を押すと、締切の入力行が増える
3. 社名だけを入力して「登録する」を押すと、一覧に企業が表示される

## 11. 制約・注意事項

- データはブラウザごとに保存されるため、端末間で同期されません。
- ブラウザの保存領域を削除すると、登録データが消える可能性があります。
- ID・パスワードのアプリ用パスワードを忘れると、復元できません。
- 企業サイトの内容を自動取得しません。
- 自動ログイン、自動応募、応募状況の自動更新は行いません。
- GitHub Pagesの反映には少し時間がかかる場合があります。
- PWAをホーム画面へ追加している場合でも、更新確認は通常のブラウザタブで行うと確実です。
