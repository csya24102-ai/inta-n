# インターン選考管理 PWA

企業ごとの応募・選考状況、複数の締切、インターン日時、面接日時、マイページURL、ID・パスワードを、ブラウザ内で管理する静的Webアプリです。

> **対象バージョン：v8**  
> この版では、選考状況・希望度の選択、**「締切を追加」**、および **「認証情報をロック解除」** ボタンが動作するよう修正しています。

---

## 1. 主な機能

### 企業情報の登録・編集・削除
- **社名のみ必須**で登録できます。
- 選考状況、希望度、締切、インターン日時、面接日時、マイページURL、ID、パスワード、メモは空欄でも登録できます。
- 同じ社名を重複登録しないように管理します。

### 選考状況・希望度の選択
選考状況の例：
- 未選択
- 応募予定
- 応募準備中
- 応募済み
- 書類選考中
- 面接予定
- 選考中
- 合格
- 不合格
- 辞退
- 終了

希望度の例：
- 未選択
- 第一志望
- 高い
- 普通
- 低い

### 複数の締切管理
1社に対して、締切を複数追加できます。

登録できる情報：
- 締切の種類  
  例：ES、Webテスト、適性検査、応募、書類提出、その他
- 締切名  
  例：「ES提出」「SPI受検」「録画面接提出」
- 締切日時

締切日時がまだ決まっていない場合は、**種類や締切名だけ**を保存できます。

### 日時管理
以下は任意で入力できます。

- インターン日時
- 面接日時
- 各種締切日時

### マイページ・公式サイトの確認
画面上部の企業名検索で、企業のサイトを開けます。

- 企業情報にマイページURLが登録済みの場合  
  → 登録済みのマイページURLを開きます。
- 企業が未登録、またはマイページURLが未登録の場合  
  → `社名 公式サイト` の検索結果を新しいタブで開きます。

> 企業名だけから「正しい公式サイト」を完全自動判定する機能はありません。検索結果から公式サイトを確認してください。

### ID・パスワード管理
- IDとパスワードを保存できます。
- ID・パスワードをワンクリックでコピーできます。
- パスワードは通常は伏字で表示されます。
- 認証情報は、アプリ用パスワードを使ってブラウザ内で暗号化して保存します。

## 2. 作らないもの

次の機能は対象外です。

- 企業情報、締切、マイページURLの自動収集
- 企業サイトへの自動ログイン
- ESや応募フォームの自動送信
- 複数端末間でのデータ同期
- 他の利用者とのデータ共有
- サーバー側データベースへの保存

## 4. 起動方法（PCで確認する場合）

### Pythonを使う場合

フォルダをVS Codeで開き、ターミナルで次を実行します。

#### Windows

```bash
py -m http.server 8000
```

`py` が使えない場合：

```bash
python -m http.server 8000
```

#### Mac

```bash
python3 -m http.server 8000
```

ブラウザで次を開きます。

```text
http://localhost:8000
```

> `index.html` を直接ダブルクリックして `file://` で開くのではなく、`localhost` 経由で開いてください。PWA機能やService Workerが正しく動かない場合があります。

## 5. 初回利用手順

1. アプリを開きます。
2. 右上の **「認証情報をロック解除」** を押します。
3. 8文字以上のアプリ用パスワードを設定します。
4. 社名を入力します。
5. 必要に応じて、選考状況・希望度・締切・日時・URL・ID・パスワード・メモを入力します。
6. **「登録する」** を押します。
7. 右側または下部の一覧に登録内容が表示されます。

> アプリ用パスワードを忘れると、保存済みのID・パスワードを復号できません。忘れないように保管してください。

---

## 6. 締切の追加方法（v8）

1. 登録フォームの **「締切」** セクションを確認します。
2. 初期状態で、締切入力行が1つ表示されます。
3. **「締切を追加」** を押します。
4. 新しい締切入力行が追加されます。
5. 締切の種類、締切名、締切日時を入力します。
6. 不要な行は、その行の **「削除」** を押します。

### 確認ポイント

- 「締切を追加」を1回押すと、入力行が1行増える
- 追加した締切ごとに、種類・名称・日時を別々に入力できる
- 締切日時が未定でも、種類や名前を保存できる
- 社名だけで登録できる

## 7. GitHub Pagesで公開する方法（Gitなし）

### 7-1. GitHubへファイルをアップロード

1. GitHubで新しいリポジトリを作成します。  
   例：`intern-selection-manager`
2. **Add file** → **Upload files** を選びます。
3. このフォルダ内のファイルを、リポジトリ直下へアップロードします。
4. **Commit changes** を押します。

正しい配置：

```text
intern-selection-manager/
├── index.html
├── styles.css
├── app.js
├── manifest.webmanifest
├── sw.js
├── README.md
└── icons/
    └── icon.svg
```

### 7-2. GitHub Pagesを有効化

1. リポジトリの **Settings** を開きます。
2. 左側メニューの **Pages** を選びます。
3. 次のように設定します。

```text
Source: Deploy from a branch
Branch: main
Folder: /(root)
```

4. **Save** を押します。
5. 表示された公開URLを開きます。

公開URLの例：

```text
https://あなたのGitHubユーザー名.github.io/intern-selection-manager/
```

## 8. 更新方法（Gitなし）

VS Codeで編集した後、GitHubのWeb画面で同名ファイルを更新します。

### 重要：v8を更新するときは3ファイルをセットで更新

以下は必ず同じ版を使ってください。

```text
index.html
app.js
sw.js
```

`index.html`だけ、または`app.js`だけを更新すると、選択肢や「締切を追加」ボタンが動かなくなることがあります。

### 更新手順

1. GitHubのリポジトリで対象ファイルを開きます。
2. 右上の鉛筆アイコン **Edit this file** を押します。
3. VS Codeの新しいファイル内容を全文コピーします。
4. GitHubの編集画面の内容を全文置き換えます。
5. **Commit changes** を押します。
6. `index.html`、`app.js`、`sw.js` を同じように更新します。

## 9. PWA更新時の重要事項

`index.html`、`styles.css`、`app.js` を更新した場合は、`sw.js` のキャッシュ名も更新してください。

v8の`sw.js`先頭は、次の状態です。

```javascript
const CACHE_NAME = 'intern-selection-manager-v8';
```

次回更新時は、数字を増やします。

```javascript
const CACHE_NAME = 'intern-selection-manager-v8';
```

更新後、公開ページは以下のようにクエリ付きで開くと、キャッシュの影響を受けにくくなります。

```text
https://あなたのGitHubユーザー名.github.io/intern-selection-manager/?v=7
```

さらに強制再読み込みをします。

| OS | 操作 |
|---|---|
| Windows | `Ctrl + Shift + R` |
| Mac | `Cmd + Shift + R` |

> ブラウザの「サイトデータを削除」や「Clear site data」は、登録済みの企業データも消える可能性があるため、必要な場合以外は実行しないでください。

## 10. 保存データとセキュリティ

### 保存場所
登録した企業情報は、利用している端末・ブラウザ内に保存されます。

そのため、次のデータは同期されません。

- PCで登録した内容をスマートフォンで確認する
- Chromeで登録した内容をSafariで確認する
- 別のPCから同じ一覧を確認する

### GitHubに送信されない情報
通常の利用では、以下の入力内容はGitHubへ送信されません。

- 社名
- 選考状況
- 希望度
- 締切
- インターン日時
- 面接日時
- マイページURL
- ID
- パスワード
- メモ

GitHubへ公開するのは、HTML・CSS・JavaScriptなどの**アプリのソースコード**です。

### 注意
- 本物のID・パスワードを`index.html`や`app.js`へ書かないでください。
- CSVファイル、バックアップファイル、画面キャプチャ、個人情報をGitHubリポジトリへアップロードしないでください。
- 共有PCや学校PCで本物の認証情報を保存しないでください。
- 大切なパスワードは、専用のパスワードマネージャーにも保管することを推奨します。

## 12. よくある問題

### 選考状況・希望度を押しても候補が出ない
`index.html`、`app.js`、`sw.js` が同じ版で更新されていない可能性があります。3ファイルをセットで更新してください。

### 「締切を追加」を押しても入力行が増えない
v8の`app.js`が反映されていない可能性があります。

次を確認してください。

```javascript
const CACHE_NAME = 'intern-selection-manager-v8';
```

また、`index.html`、`app.js`、`sw.js` の3ファイルをセットで更新し、公開ページを強制再読み込みしてください。

### 更新後も古い画面が出る
PWAキャッシュが残っている可能性があります。

1. `sw.js` のキャッシュ名を次の版番号へ上げます。
2. GitHubで`sw.js`を更新します。
3. 公開URLに `?v=8` のような文字を付けて開きます。
4. 強制再読み込みします。

### 登録データが消えた
ブラウザのサイトデータ削除、シークレットモード、別ブラウザ・別端末の利用が原因として考えられます。端末・ブラウザ内の保存データは自動同期されません。

## 8. 更新方法（Gitなし）

VS Codeで編集した後、GitHubのWeb画面で同名ファイルを更新します。

### 重要：v7を更新するときは3ファイルをセットで更新

以下は必ず同じ版を使ってください。

```text
index.html
app.js
sw.js
```

`index.html`だけ、または`app.js`だけを更新すると、選択肢や「締切を追加」ボタンが動かなくなることがあります。

### 更新手順

1. GitHubのリポジトリで対象ファイルを開きます。
2. 右上の鉛筆アイコン **Edit this file** を押します。
3. VS Codeの新しいファイル内容を全文コピーします。
4. GitHubの編集画面の内容を全文置き換えます。
5. **Commit changes** を押します。
6. `index.html`、`app.js`、`sw.js` を同じように更新します。

## 9. PWA更新時の重要事項

`index.html`、`styles.css`、`app.js` を更新した場合は、`sw.js` のキャッシュ名も更新してください。

v7の`sw.js`先頭は、次の状態です。

```javascript
const CACHE_NAME = 'intern-selection-manager-v7';
```

次回更新時は、数字を増やします。

```javascript
const CACHE_NAME = 'intern-selection-manager-v8';
```

更新後、公開ページは以下のようにクエリ付きで開くと、キャッシュの影響を受けにくくなります。

```text
https://あなたのGitHubユーザー名.github.io/intern-selection-manager/?v=7
```

さらに強制再読み込みをします。

| OS | 操作 |
|---|---|
| Windows | `Ctrl + Shift + R` |
| Mac | `Cmd + Shift + R` |

> ブラウザの「サイトデータを削除」や「Clear site data」は、登録済みの企業データも消える可能性があるため、必要な場合以外は実行しないでください。

## 10. 保存データとセキュリティ

### 保存場所
登録した企業情報は、利用している端末・ブラウザ内に保存されます。

そのため、次のデータは同期されません。

- PCで登録した内容をスマートフォンで確認する
- Chromeで登録した内容をSafariで確認する
- 別のPCから同じ一覧を確認する

### GitHubに送信されない情報
通常の利用では、以下の入力内容はGitHubへ送信されません。

- 社名
- 選考状況
- 希望度
- 締切
- インターン日時
- 面接日時
- マイページURL
- ID
- パスワード
- メモ

GitHubへ公開するのは、HTML・CSS・JavaScriptなどの**アプリのソースコード**です。

### 注意
- 本物のID・パスワードを`index.html`や`app.js`へ書かないでください。
- CSVファイル、バックアップファイル、画面キャプチャ、個人情報をGitHubリポジトリへアップロードしないでください。
- 共有PCや学校PCで本物の認証情報を保存しないでください。
- 大切なパスワードは、専用のパスワードマネージャーにも保管することを推奨します。

## 12. よくある問題

### 選考状況・希望度を押しても候補が出ない
`index.html`、`app.js`、`sw.js` が同じ版で更新されていない可能性があります。3ファイルをセットで更新してください。

### 「締切を追加」を押しても入力行が増えない
v7の`app.js`が反映されていない可能性があります。

次を確認してください。

```javascript
const CACHE_NAME = 'intern-selection-manager-v7';
```

また、`index.html`、`app.js`、`sw.js` の3ファイルをセットで更新し、公開ページを強制再読み込みしてください。

### 更新後も古い画面が出る
PWAキャッシュが残っている可能性があります。

1. `sw.js` のキャッシュ名を次の版番号へ上げます。
2. GitHubで`sw.js`を更新します。
3. 公開URLに `?v=8` のような文字を付けて開きます。
4. 強制再読み込みします。

### 登録データが消えた
ブラウザのサイトデータ削除、シークレットモード、別ブラウザ・別端末の利用が原因として考えられます。端末・ブラウザ内の保存データは自動同期されません。

- ID・パスワードはアプリ用パスワードでAES-GCM暗号化してブラウザ内に保存
- PWAとしてインストール、オフラインでアプリ画面を起動

## 非対象
- 企業名だけからマイページURLを自動検索する機能# インターン選考管理PWA（社名のみ必須版）

## 主な変更点

- 卒業年度の入力・表示を削除
- **社名だけで登録可能**。選考状況、希望度、締切、インターン日時、URL、ID、パスワード、メモはすべて任意
- 締切を複数登録可能
  - ES締切
  - Webテスト締切
  - 適性検査締切
  - 応募締切
  - 書類提出締切
  - その他（締切名を自由入力）
- インターン日時を任意で入力可能
- 登録済みでURLがある企業はマイページを開く。未登録・URL未登録企業は「社名 公式サイト」の検索結果を開く
