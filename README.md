# インターン選考管理 PWA（v10）

## 今回の修正内容

v10では、次の問題をまとめて修正しています。

- 右上の表示が「確認中」のままになる
- 「認証情報をロック解除」を押しても反応しない
- 「締切を追加」を押しても入力行が増えない
- GitHub Pages更新後に、卒業年度などの古い画面が出続ける

この版は、JavaScriptの起動時にネットワーク表示を最初に更新し、締切追加ボタンのイベントを直接登録しています。また、Service Workerをネットワーク優先に変更しています。

## 重要：GitHubで更新するファイル

以下を**必ずセットで**更新してください。

```text
index.html
styles.css
app.js
sw.js
manifest.webmanifest
reset.html
icons/icon.svg
```

特に `index.html` / `app.js` / `sw.js` の3つだけ新旧が混ざると、画面とボタン処理が一致しません。

## 更新後に最初にすること

1. GitHub Pagesの公開URLを通常のChromeタブで開きます。
2. 右上の「更新が反映されない場合」を押します。
3. 「古いPWAキャッシュを削除して最新版を開く」を1回押します。
4. 元の画面に戻り、右上が「オンライン」または「オフライン」になったことを確認します。

この更新ページが削除するのはService WorkerとCache Storageだけです。企業データ・ID・パスワードを保持するlocalStorageは削除しません。

## 公開URLの例

```text
https://あなたのGitHubユーザー名.github.io/リポジトリ名/?v=10
```

## v10の確認項目

- 右上が「確認中」ではなく「オンライン」または「オフライン」になる
- 「認証情報をロック解除」でパスワード入力ダイアログが開く
- 「締切を追加」で締切の入力行が1行増える
- 社名だけで登録できる
- 卒業年度の入力欄がない
- 未登録の会社名を検索すると「社名 公式サイト」の検索が開く
